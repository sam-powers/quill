# Quill v0.2.0

**The document editor that can hold a conversation — now with any document.**

In v0.1.0 the conversation belonged to documents that Claude Code wrote. This release removes that requirement: any Markdown file — one someone sent you, or one you wrote yourself — can hold a conversation of its own.

## Highlights

- **Start new session** — the session picker gains a **Start new session** button for documents no Claude session wrote. Save the document, mention `@claude` in a comment, and choose it: Quill creates a fresh Claude session in the document's folder on the first request and resumes that same session for every comment after, so the conversation accumulates context just like an authoring session. Prompts to these sessions never claim Claude wrote the document, and they always carry the full current text.
- **Comment threads read top-down** — your question now always renders above Claude's streaming answer in the thread.
- **Session picker polish** — the "no sessions found" state now points you at Start new session instead of dead-ending.

Documents and sidecars are fully compatible with v0.1.0 in both directions.

## Install

| Platform              | Asset           |
| --------------------- | --------------- |
| macOS (Apple Silicon) | `…_aarch64.dmg` |
| macOS (Intel)         | `…_x64.dmg`     |

Builds are **unsigned**: right-click **Quill.app** → **Open** → **Open** on first launch.

The `@claude` features require the [Claude Code](https://claude.com/claude-code) CLI on the same machine. See the [User Guide](https://github.com/sam-powers/quill/blob/main/docs/USER_GUIDE.md) for a walkthrough.

## Known limitations

- macOS only for now. The `@claude` integration finds the Claude CLI via Unix paths; rather than ship a partial experience, Windows and Linux installers are deferred (both platforms can [build from source](https://github.com/sam-powers/quill#building-from-source)).
- No code signing yet (Gatekeeper warning above).
- Single document per window; no multi-user collaboration or cloud sync.
- Suggesting mode tracks insertions and deletions (a replacement is a delete + insert pair).
