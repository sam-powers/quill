# Quill v1.0.0

**The document editor that can hold a conversation — now 1.0.**

Quill opens a Markdown file and gives it what Google Docs gave the .doc: suggesting mode, margin comments — and a collaborator. Link a document to a Claude session and the conversation happens _in the margins_: ask questions in comment threads, request rewrites that arrive as Accept/Reject tracked changes, and now hand over the whole draft for a final polish. V1 rounds out the everyday editor underneath it: find & replace, crash recovery, link editing, faithful round-tripping of real-world Markdown.

## Highlights

- **Review full document** — a ✨ button at the top of the comment column sends the whole draft to Claude in one pass. Tell it what to look for (a sensible review prompt is pre-filled — edit it to "make it 20% shorter" or "check the tone for a customer audience"), choose whether you want **comments**, **suggestions**, or both, and watch the assessment stream in. Claude's notes land as margin comments anchored to the passages they're about; its edits land as tracked changes you Accept or Reject one by one. Nothing touches your document without your approval.
- **Find & replace (Cmd+F)** — with match stepping, Replace, and Replace All (one undo step). In Suggesting mode, replacements become tracked changes like any other edit.
- **Crash recovery** — while a document has unsaved changes, Quill snapshots it (content, comments, and suggestions included) every few seconds. If the app or your Mac goes down, the next launch offers to restore exactly where you were.
- **Link editing (Cmd+K)** — add, edit, and remove links from a small inline editor instead of hand-writing Markdown.
- **Real-world Markdown round-trips faithfully** — tables, task lists, and images survive open → edit → save byte-stable. Relative image paths display correctly, and Quill warns you on open if a file contains constructs it can't preserve (footnotes, raw HTML) _before_ you save over them.
- **Everyday-app polish** — File → Open Recent, window size/position remembered across launches, native spellcheck squiggles, and comment cards that stay glued to their text at any zoom level.

Documents and sidecars are fully compatible with every 0.x release in both directions.

## Install

| Platform              | Asset           |
| --------------------- | --------------- |
| macOS (Apple Silicon) | `…_aarch64.dmg` |
| macOS (Intel)         | `…_x64.dmg`     |

Builds are **unsigned**: right-click **Quill.app** → **Open** → **Open** on first launch.

The `@claude` features require the [Claude Code](https://claude.com/claude-code) CLI on the same machine. See the [User Guide](https://github.com/sam-powers/quill/blob/main/docs/USER_GUIDE.md) for a walkthrough.

## Known limitations

- macOS only for now. The `@claude` integration finds the Claude CLI via Unix paths; rather than ship a partial experience, Windows and Linux installers are deferred (both platforms can [build from source](https://github.com/sam-powers/quill#building-from-source)).
- No code signing yet (Gatekeeper warning above).
- Single document per window; no multi-user collaboration or cloud sync.
