# Quill v0.1.0

First public release. Quill is a desktop Markdown editor for **reviewing and revising prose** — Google Docs' suggesting mode, but for local `.md` files, with an AI reviewer built in.

## Highlights

- **Suggesting mode** — edits become tracked insertions/deletions with per-change Accept / Reject cards in the margin, plus Accept All / Reject All.
- **Inline comments** — threaded, anchored to text ranges, with resolve/unresolve and collision-free margin cards.
- **`@claude` replies** — link a document to the Claude Code session that authored it and ask questions in a comment thread; answers stream back inline, compaction-aware (line diff when the session remembers the doc, full document when it doesn't).
- **AI-authored tracked changes** — ask Claude to revise ("tighten this", "fix the grammar") and its edits land as tracked changes attributed to Claude, reviewed exactly like a human's. Scope follows your phrasing: highlight → paragraph → whole document.
- **Reference folders** — link a folder of source material; every `@claude` request grants read access plus a file manifest, so Claude can check your prose against your sources.
- **Portable persistence** — documents stay plain `.md`; review data rides in a `<name>.comments.json` sidecar that's removed when empty. Corrupt sidecars are never overwritten.
- **Desktop niceties** — native File menu, unsaved-changes guard everywhere (close, quit, deep links), `quill://open?file=…` deep links, a Claude Code plugin (`/quill-integration:open-in-quill`), zoom, four themes.

## Install

| Platform              | Asset           |
| --------------------- | --------------- |
| macOS (Apple Silicon) | `…_aarch64.dmg` |
| macOS (Intel)         | `…_x64.dmg`     |

Builds are **unsigned**: right-click **Quill.app** → **Open** → **Open** on first launch.

The `@claude` features require the [Claude Code](https://claude.com/claude-code) CLI on the same machine. See the [User Guide](https://github.com/sam-powers/quill/blob/main/docs/USER_GUIDE.md) for a walkthrough.

## Known limitations

- macOS only for now. The `@claude` integration finds the Claude CLI via Unix paths; rather than ship a partial experience, Windows and Linux installers are deferred (both platforms can [build from source](https://github.com/sam-powers/quill#building-from-source)).
- No code signing yet (Gatekeeper warning above).
- Single document per window; no multi-user collaboration or cloud sync.
- Suggesting mode tracks insertions and deletions (a replacement is a delete + insert pair).
